Llave primaria (PK):

Es el identificador de columna de la tabla en nos encontramos. Es un valor único que hace una distinción entre cada fila de la tabla especificada. No puede repetirse en la misma tabla.


Llave foránea (FK):

Es una llave primaria (PK) que pertenece a otra tabla, pero que la usamos para relacionar las tablas. Utilizamos las claves principales como externas en nuestra tabla de hechos. Estás pueden repetirse.


En las tablas dim la información no se repite, es única.


En las tablas de hecho si puede repetirse el número de cliente por ejemplo (FK) pero no el número de factura (PK)

Funciones modificadoras del contexto de filtro en el CALCULATE:


Tener en cuenta: Los filtros del CALCULATE solo aceptan booleanos y funciones de tabla. Internamente, CALCULATE convierte todos los filtros en tablas.


1-Para modificar filtros:

REMOVEFILTERS(): Borrar filtros de las tablas o columnas especificadas. Funciona idéntico al ALL(), pero solo puede ser usado como modificador del CALCULATE.

REMOVEFILTERS([table | column[, column[, column[,…]]]])


KEEPFILTERS(): Modifica cómo se aplican los filtros al evaluar una función CALCULATE o CALCULATETABLE. No modifica los filtros existentes sino que suma un nuevo contexto de filtros. Cuando usa KEEPFILTERS, cualquier filtro existente en el contexto actual se compara con las columnas en los argumentos de filtro, y la intersección de esos argumentos se usa como contexto para evaluar la expresión. El efecto sobre cualquier columna es que se aplican ambos conjuntos de argumentos: tanto los argumentos de filtro usados en CALCULATE como los filtros en los argumentos de la función KEEPFILTER. En otras palabras, mientras que los filtros CALCULATE reemplazan el contexto actual, KEEPFILTERS agrega filtros al contexto actual.

KEEPFILTERS(expression)


ALL(): Devuelve todas las filas de una tabla o todos los valores de una columna, ignorando cualquier filtro que se haya aplicado. Esta función no se usa sola, sino que sirve como una función intermedia que se puede usar para cambiar el conjunto de resultados sobre los que se realiza algún otro cálculo. ALL solo acepta como argumentos tablas físicas. Se usa cuando necesitamos ver valores sin filtrar y no queremos que el contexto de filtro actúe.

ALL( [table o column[, column[, column[,…]]]] )


FILTER(): Devuelve una tabla que representa un subconjunto de otra tabla o expresión. Puede usar FILTER para reducir el número de filas en la tabla con la que está trabajando y usar solo datos específicos en los cálculos. FILTER no se usa de forma independiente, sino como una función que está incrustada en otras funciones que requieren una tabla como argumento.

FILTER(table,filter)


DISTINCT(): Devuelve una tabla de una columna que contiene los distintos valores de la columna especificada. En otras palabras, los valores duplicados se eliminan y solo se devuelven valores únicos. Tener en cuenta: La función VALUES() es similar a DISTINCT(); también se puede utilizar para devolver una lista de valores únicos y, por lo general, devolverá exactamente los mismos resultados que DISTINCT. Sin embargo, en algún contexto, VALUES siempre muestra una fila en blanco por si hay valores “de más” en alguna de las tablas.

DISTINCT(column)


VALUES(): Cuando el parámetro de entrada es un nombre de columna, devuelve una tabla de una columna que contiene los distintos valores de la columna especificada. Los valores duplicados se eliminan y solo se devuelven valores únicos. Se agrega un valor EN BLANCO. Cuando el parámetro de entrada es un nombre de tabla, devuelve las filas de la tabla especificada. Las filas duplicadas se conservan. Se agrega una fila EN BLANCO. Esta función no se puede usar para devolver valores a una celda o columna en una hoja de trabajo; más bien, lo usa como una función intermedia, anidada en una fórmula, para obtener una lista de valores distintos que se pueden contar o usar para filtrar o sumar otros valores. Cuando usa la función en un contexto que ha sido filtrado, los valores únicos devueltos por VALUES se ven afectados por el filtro.

VALUES(TableNameOrColumnName)


SELECTVALUE(): Devuelve el valor cuando el contexto de columnName se ha filtrado a un solo valor distinto. De lo contrario, devuelve resultado alternativo.

SELECTEDVALUE(columnName[, alternateResult])


ALLEXCEPT(): Elimina todos los filtros de contexto de la tabla, excepto los que se han aplicado a las columnas especificadas. El primer argumento de la función ALLEXCEPT debe ser una referencia a una tabla base. Todos los argumentos posteriores deben ser referencias a columnas base. No puede usar expresiones de tabla o expresiones de columna con la función ALLEXCEPT. Esta función no se usa sola, sino que sirve como una función intermedia que se puede usar para cambiar el conjunto de resultados sobre los que se realiza algún otro cálculo. Conveniente para situaciones en las que desea eliminar los filtros en muchas columnas de una tabla, pero no en todas.

ALLEXCEPT(table, column[,column[,…]])


ALLSELECTED(): Elimina los filtros de contexto de las columnas y filas de la consulta actual, al tiempo que conserva todos los demás filtros de contexto o filtros explícitos. La función ALLSELECTED obtiene el contexto que representa todas las filas y columnas de la consulta, al tiempo que mantiene filtros y contextos explícitos distintos de los filtros de fila y columna. Esta función se puede utilizar para obtener totales visuales en las consultas. Si hay un argumento, el argumento es tableName o columnName . Si hay más de un argumento, deben ser columnas de la misma tabla. Esta función es diferente de ALL() porque conserva todos los filtros establecidos explícitamente dentro de la consulta y conserva todos los filtros de contexto que no sean los filtros de fila y columna.

ALLSELECTED([tableName | columnName[, columnName[, columnName[,…]]]] )


SELECTCOLUMNS(): Retorna una tabla con el mismo número de filas que la tabla especificada como primer argumento. La tabla devuelta tiene una columna para cada par de argumentos name, scalar_expression, y cada expresión se evalúa en el contexto de una fila del argumento table especificado. SELECTCOLUMNS tiene la misma firma que ADDCOLUMNS y tiene el mismo comportamiento excepto que en lugar de comenzar con la tabla especificada, SELECTCOLUMNS comienza con una tabla vacía antes de agregar columnas.

SELECTCOLUMNS(table, name, scalar_expression [, name, scalar_expression]…)


ADDCOLUMNS(): Agrega columnas calculadas a la tabla o expresión de tabla dada.

ADDCOLUMNS(table, name, expression[, name, expression]…)


SUMMARIZE(): Devuelve una tabla de resumen de los totales solicitados en un conjunto de grupos. Cada columna para la que defina un nombre debe tener una expresión correspondiente; de lo contrario, se devuelve un error. El primer argumento, nombre, define el nombre de la columna en los resultados. El segundo argumento, expresión, define el cálculo realizado para obtener el valor de cada fila de esa columna.

SUMMARIZE (table, groupBy_columnName[, groupBy_columnName]…[, name, expression]…)

GIT


GIT init: Se usa para iniciar un repositorio en una carpeta.

GIT add . : Se usa para confirmar que vamos a guardar todos los elementos de la carpeta en el repositorio.

GIT add . nombrearchivo: Esto es útil en caso de que quisiramos almacenar en el repositorio solo un archivo especifico.

GIT commit -m "Comentario o descripcion del commit a realizar": Se usa para confirmar el almacenamiento en el repositorio.

GIT log --oneline: Se usa para saber el historial de todos nuestros commits.

GIT status -s: Se utiliza para saber el status de todos nuestros archivos en el repositorio, si no muestra nada es porque ya fueron guardados todos en el repositorio (Comiteados).

GIT reset --hard (Código de Commit): Se usa para volver a un commit en particular.

GIT branch: Se usa para conocer todas las ramas disponibles en el repositorio.

GIT branch (Nombre de la rama que deseamos crear): Se usa para crear nuevas ramas.

GIT checkout (Nombre de la rama a la que queremos pasar): Se usa para intercambiar de una rama a otra.

GIT branch -d rama2 <- (nombre de la rama a borrar): Este comando se usa para borrar ramas.

GIT tag (Nombre del Tag) -m "Comentario": Se usa para crear un Tag.

GIT push --tags: Se usa para confirmar el archivo final subido en Github, si accedemos a ese Tag podremos descargar un comprimido con todos los archivos.
