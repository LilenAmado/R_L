Power BI


Llave primaria (PK): Es el identificador de columna de la tabla en nos encontramos. Es un valor único que hace una distinción entre cada fila de la tabla especificada. No puede repetirse en la misma tabla.

Llave foránea (FK): Es una llave primaria (PK) que pertenece a otra tabla, pero que la usamos para relacionar las tablas. Utilizamos las claves principales como externas en nuestra tabla de hechos. Estás pueden repetirse.

En las tablas dim la información no se repite, es única.
En las tablas de hecho si puede repetirse el número de cliente por ejemplo (FK) pero no el número de factura (PK)

GIT


GIT init: Se usa para iniciar un repositorio en una carpeta.

GIT add . : Se usa para confirmar que vamos a guardar todos los elementos de la carpeta en el repositorio.

GIT add . nombrearchivo: Esto es útil en caso de que quisiramos almacenar en el repositorio solo un archivo especifico.

GIT commit -m "Comentario o descripcion del commit a realizar": Se usa para confirmar el almacenamiento en el repositorio.

GIT log --oneline: Se usa para saber el historial de todos nuestros commits.

GIT status -s : Se utiliza para saber el status de todos nuestros archivos en el repositorio, si no muestra nada es porque ya fueron guardados todos en el repositorio (Comiteados).

GIT reset --hard (Código de Commit): Se usa para volver a un commit en particular.

GIT branch: Se usa para conocer todas las ramas disponibles en el repositorio.

GIT branch (Nombre de la rama que deseamos crear): Se usa para crear nuevas ramas.

GIT checkout (Nombre de la rama a la que queremos pasar): Se usa para intercambiar de una rama a otra.

GIT branch -d rama2 <- (nombre de la rama a borrar): Este comando se usa para borrar ramas.

GIT tag (Nombre del Tag) -m "Comentario": Se usa para crear un Tag.

GIT push --tags: Se usa para confirmar el archivo final subido en Github, si accedemos a ese Tag podremos descargar un comprimido con todos los archivos.
