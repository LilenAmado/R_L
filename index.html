Power BI


Llave primaria (PK):

Es el identificador de columna de la tabla en nos encontramos. Es un valor único que hace una distinción entre cada fila de la tabla especificada. No puede repetirse en la misma tabla.


Llave foránea (FK):

Es una llave primaria (PK) que pertenece a otra tabla, pero que la usamos para relacionar las tablas. Utilizamos las claves principales como externas en nuestra tabla de hechos. Estás pueden repetirse.


En las tablas dim la información no se repite, es única.


En las tablas de hecho si puede repetirse el número de cliente por ejemplo (FK) pero no el número de factura (PK)

Funciones Math & Stats:


SUM(): Suma todos los valores de una columna.

SUM(Columna)


AVERAGE(): Devuelve el promedio de todos los números de una columna

AVERAGE(Columna)


MAX(): Devuelve el valor más grande en una columna o entre dos expresiones escalares

MAX(Columna)

MAX(Expresión 1, Expresión 2])

MAX(5*4, 5*5)


MIN(): Devuelve el valor más pequeño en una columna o entre dos expresiones escalares. Mismo que MAX

MIN(Columna)

MIN(Expresión 1, Expresión 2])


DIVIDE(): Es mejor que utilizar el “/” porque si hay un error se puede poner un valor alternativo.

DIVIDE(Numerador, Denominador, Resultado Alternativo)


COUNT(): Cuenta todos los números de filas que hay en una columna que contiene números.

COUNT(Columna)


COUNTA(): Cuenta el número de valores en una columna. En blanco no los cuenta.

COUNTA(Columna)


COUNTROWS(): Cuenta el número de filas en la tabla de una columna específica o una tabla definida por una expresión. Cuenta los registros en blanco también.

COUNTROWS(Tabla)


COUNTBLANK(): Cuenta todas las filas que estan en blanco

COUNTBLANK(Columna)


DISTINCTCOUNT(): Cuenta el número de valores distintos o únicos en una columna

DISTINCTCOUNT(Columna)

No repite los resultados iguales

GIT


GIT init: Se usa para iniciar un repositorio en una carpeta.

GIT add . : Se usa para confirmar que vamos a guardar todos los elementos de la carpeta en el repositorio.

GIT add . nombrearchivo: Esto es útil en caso de que quisiramos almacenar en el repositorio solo un archivo especifico.

GIT commit -m "Comentario o descripcion del commit a realizar": Se usa para confirmar el almacenamiento en el repositorio.

GIT log --oneline: Se usa para saber el historial de todos nuestros commits.

GIT status -s: Se utiliza para saber el status de todos nuestros archivos en el repositorio, si no muestra nada es porque ya fueron guardados todos en el repositorio (Comiteados).

GIT reset --hard (Código de Commit): Se usa para volver a un commit en particular.

GIT branch: Se usa para conocer todas las ramas disponibles en el repositorio.

GIT branch (Nombre de la rama que deseamos crear): Se usa para crear nuevas ramas.

GIT checkout (Nombre de la rama a la que queremos pasar): Se usa para intercambiar de una rama a otra.

GIT branch -d rama2 <- (nombre de la rama a borrar): Este comando se usa para borrar ramas.

GIT tag (Nombre del Tag) -m "Comentario": Se usa para crear un Tag.

GIT push --tags: Se usa para confirmar el archivo final subido en Github, si accedemos a ese Tag podremos descargar un comprimido con todos los archivos.

PASOS PARA CLONAR UN REPO, MODIFICARLO Y LUEGO SUBIRLO DE NUEVO AL REPOSITORIO DE GITHUB

1) GIT clone (link del repo): Se usa para clonar un repo en GitHub en la maquina local y trabajar con esos archivos

2) Una vez realizado el paso anterior lo que procedemos a realiazr es abrir una nueva consola en el repositorio local

3) hacemos un GIT add .

4) un Commit

5) Finalmente usamos este comando

GIT push (link del repo)

PASOS PARA MERGEAR UN REPOSITORIO

1) Pasar a la Rama Master con GIT checkout

2) Desde la Rama Master hacer un GIT merge (nombre de la rama a la queremos mergear con la original)
